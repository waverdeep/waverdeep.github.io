실시간 이상탐지 AI 모니터링 시스템

시스템 지표(CU 등)의 이상을 사전 탐지하는 End-to-end 모니터링 파이프라인. Prometheus로 수집한 시계열을 N-HiTS 예측 모델로 분석하고, Grafana 시각화와 Google Chat 실시간 알림까지 연결. 평균 10분 이내로 장애를 사전 예측하며, 웹보드 6개 서비스와 브라운더스트2에 적용되어 DC팀(24시간 모니터링)의 실운영 도구로 채택됨.

2022.09 — 운영 중 AI 모델 연구·개발 · 시스템 구축 평균 ~10분 이내 예측 웹보드 6개 + 브라운더스트2 적용

02 Problem

실시간 서비스는 트래픽·시스템 지표 변동에 민감하여, 작은 장애도 사용자 경험과 매출에 직접 영향을 줍니다. 기존에는 모니터링 담당자가 다수의 시계열 대시보드를 직접 눈으로 확인하는 방식이었고, 데이터 양이 많아 놓치거나 인지가 늦어지는 케이스가 다수 발생했습니다.

"사람이 그래프를 보고 판단하기 전에, 모델이 먼저 이상을 감지해서 알려주자."

프로젝트 인수 시점에 이미 3계층 Linear 모델 기반의 탐지기가 존재했지만, trend / seasonality / cycle이 얽힌 실제 트래픽 패턴에는 취약했고, 평상시 정상 변동을 이상으로 오탐하거나 실제 장애를 놓치는 경우가 많았습니다.

03 Approach

N-HiTS로 모델 교체 — trend / seasonality / cycle 자동 분해

기존 3계층 Linear 모델을 N-HiTS(Darts 구현체)로 교체했습니다. N-HiTS는 다중 해상도의 시계열 분해를 내장하고 있어, trend · seasonality · cycle을 명시적 feature engineering 없이도 학습합니다. 실제 운영 데이터 회귀 테스트에서 기존 모델이 놓친 장애를 N-HiTS가 성공적으로 탐지하는 케이스를 다수 확인했습니다.

End-to-end 모니터링 파이프라인

모델만 좋아서는 실운영으로 이어지지 않기 때문에, 수집부터 알림까지 닫힌 루프를 설계했습니다.

  • 수집 — Prometheus가 5분 간격으로 CU(Concurrent User) 등 핵심 지표를 수집.
  • 예측 — N-HiTS 모델이 향후 구간을 예측하고, 실측과의 편차로 이상 점수를 산출.
  • 시각화 — Grafana 대시보드에 예측선·실측선·편차 밴드를 함께 표시.
  • 알림 — Google Chat 실시간 알림 + 10분 간격 브리핑. 담당자 온콜 동선에 자연스럽게 맞춤.

운영 편의 기능 — "쓰이지 않는 AI"가 되지 않도록

  • 서비스별 threshold 직접 조정 — 서비스마다 정상 변동 폭이 달라 획일 임계치는 위험. DC팀이 자체적으로 조정 가능하도록 UI 제공.
  • 모니터링 분류 세분화 — PC / 모바일 / 전체를 분리하여 디바이스별 패턴 차이를 반영.
  • 반복 알림 방지 알고리즘 — 동일 이상 상태에 대한 알림 폭주를 억제하여 알림 피로 감소.
  • 외부요인 시각화 — 이벤트·점검·프로모션 등 외부 요인이 있을 때, 예측 편차가 해당 요인 때문인지 한눈에 확인 가능.

05 Impact

  • 평균 10분 이내 장애 사전 예측 — 최소 5분 ~ 30분 범위에서, 실측 장애 인지보다 앞서 감지.
  • 웹보드 6개 서비스 + 브라운더스트2 전체 모니터링 적용 — 뉴맞고 · 포커 · 섯다 · 뉴베가스 · 뉴바둑 · 뉴장기.
  • N-HiTS가 기존 Linear 모델이 놓친 장애를 성공적으로 탐지 — 회귀 테스트로 교체 결정을 데이터 기반으로 수행.
  • DC팀(24시간 모니터링)의 실운영 도구로 채택 — AI 결과를 "참고용"으로 두는 게 아니라 온콜 의사결정의 1차 신호로 사용.
  • 인프라팀 OpenSearch 기반으로 확장 운영 — 초기 파이프라인이 조직의 다른 모니터링 체계로 이어짐.

06 Takeaways

  • 모델 교체(N-HiTS)의 임팩트는 "숫자"가 아니라 "놓친 장애를 잡았다"는 사례에서 나옴 — 지표 기반 비교도 중요하지만, 실제 온콜이 신뢰하게 된 결정적 순간은 과거 장애 재현 테스트였음.
  • "AI 예측"보다 "알림 동선"이 채택의 병목 — Google Chat 실시간 알림 + 10분 브리핑 + 반복 알림 억제까지 세팅되자 DC팀이 실제로 쓰기 시작. 모델 정확도를 올리는 것보다 이 동선 설계에 시간을 쓴 것이 ROI가 더 컸음.
  • threshold를 운영팀이 직접 조정하게 한 것이 수용성을 크게 높임 — AI가 블랙박스로 느껴지지 않게 "조절 가능한 레버"를 운영팀 손에 쥐여준 순간부터 AI-운영 간 신뢰 관계가 성립.
  • 외부요인 시각화는 "오탐"을 "설명 가능한 변동"으로 바꾸는 장치 — 이벤트/점검 같은 외부 맥락이 함께 보이자, 예측 편차를 보는 대화가 "틀렸다/맞았다"에서 "왜 벌어졌다"로 옮겨감.

07 Transferable Value

시계열 데이터가 존재하는 모든 도메인 — 제조 설비 예지 보전, 금융 이상거래 탐지, IoT 센서 모니터링 등에 동일한 방법론을 이식할 수 있습니다. 모델(N-HiTS/Darts)뿐 아니라, "수집 → 예측 → 시각화 → 알림 동선 → 운영팀이 만질 수 있는 레버"라는 엔드투엔드 설계 패턴이 그대로 재사용 가능합니다.
